# Gemini CLI Rules

Specific instructions for the **Gemini CLI** agent.

## Working Context
- This project follows the FMG Repository Development Bible.
- Prioritize using `grep_search` to navigate the code.
- Always check `git status` before every commit.

## Workflow
1. Research -> 2. Strategy -> 3. Execution (Plan-Act-Validate).

## Restrictions
- Do not modify files in `docs/` unless documenting a feature.
- Keep `CHANGELOG.md` updated with every `feat` or `fix`.
- Ensure `VERSION` matches `Cargo.toml`.
